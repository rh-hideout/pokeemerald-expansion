## Environment & pins

- Base: `rh-hideout/pokeemerald-expansion` @ tag **`expansion/1.16.2`** (latest stable;
  branch `main`, upstream remote named `upstream`, no push remote configured).
- Toolchain: **Arm GNU Toolchain 14.2.Rel1** (darwin-arm64) at
  `~/tools/arm-gnu-toolchain-14.2.rel1-darwin-arm64-arm-none-eabi`.
  *Why not brew:* brew's `arm-none-eabi-gcc` ships **no newlib** (no `<string.h>`),
  which this codebase requires — upstream CI installs `libnewlib-arm-none-eabi`
  explicitly. brew has no newlib package, so we use ARM's official build (which
  bundles it). No sudo required; devkitPro not needed (Makefile:69–86 supports any
  system `arm-none-eabi-` toolchain).
- Emulator (verification): **mGBA 0.10.5 built from source** (SDL + scripting) at
  `~/tools/mgba-build`, used as **libmgba** by our runner. *Why:* brew's macOS mGBA
  is Qt-only with no `--script` CLI, and 0.10.x's SDL frontend has no Lua loading
  either — so verification links libmgba directly (see below). brew's `mgba`
  formula stays installed: `mGBA.app` is the convenient way to play builds by hand.
  (Note: that formula is deprecated upstream — Qt5 EOL, disable date 2027-05-19 —
  which doesn't affect our source-built pin.)
- Emulator (upstream tests): repo-bundled `tools/mgba/mgba-rom-test-mac`
  (universal binary, runs natively on arm64).
- Env bootstrap: `. tools/habitat/env.sh` (toolchain PATH, coreutils gnubin for the
  test harness's `stdbuf`, `HH_MGBA` pointer).

## Headless verification (`make verify`)

`tools/habitat/verify.sh` builds the ROM, then runs **hh-runner**
(`tools/habitat/hh_runner.c`, ~200 lines of C linked against libmgba): no window,
no throttle, ~25 s wall for a full boot+interact pass. It never pixel-guesses —
game state is observed by reading symbols extracted from `pokeemerald.map`:

1. **boot** — taps SELECT; the expansion's quickstart config
   (`ENABLE_QUICKSTART`, on by default in dev builds) starts a new game straight
   from the title. Screenshot: last lit pre-overworld frame (the title screen).
2. **settle** — waits until `gMain.callback2 == CB2_Overworld` holds for 480
   consecutive frames (the truck sequence fades from black slowly). Screenshot.
3. **walk/face/press** — closed-loop navigation to tile (1,1) using live position
   reads from `*gSaveBlock1Ptr`, face the (1,0) sign, press A. Screenshot of the
   textbox.

Result: `HH_RESULT PASS interacted with test spot (frame 1524)`; screenshots in
`verify-out/` (gitignored — they contain Nintendo-derived art, so they stay out
of the repo). `make verify-boot` runs the same flow minus the interaction.

This runner is the seed of M1's playthrough-script system: it already has key
injection, frame-exact stepping, and symbol-driven memory assertions; libmgba
also exposes RTC override hooks we'll want for `COND_TIME_OF_DAY` testing.

## Save-region reclamation (spec §5 validation)

Measured from the real headers with the ROM's own ABI flags
(`tools/habitat/save_sizes.sh`; compile-time probes, nothing modified):

| Quantity | Bytes | Notes |
|---|---|---|
| SaveBlock1 occupancy | **15,568 / 15,872** | sectors 1–4 × 3,968 B → **304 B headroom today** |
| SaveBlock2 occupancy | 3,884 / 3,968 | 84 B headroom |
| **SaveBlock3 occupancy** | **4 / 1,624** | expansion's extension region (116 B/sector × 14) → **1,620 B free now** |
| Secret Bases slab (SB1 @ 0x1A9C) | **3,200** | 20 × 160 B — **spec §5's 3.2 KB assumption is exact** |
| Decoration arrays (SB1) | 176 | Secret-Base-adjacent, same cut |
| Contest winners (SB1) | 416 | backup target |
| TV shows + PokéNews (SB1) | 900 + 64 | bonus cut-content targets |
| Battle Frontier (SB2) | 2,272 | backup target |
| `FREE_*` config space (documented in `include/config/save.h`) | 2,516 (SB1) + 1,274 (SB2) | flip-a-flag reclamation, incl. 1,104 B ramScript + 876 B Mystery Gift |

**Analysis vs. §5's ~810 B draft allocation:**

- Spec's assumption validated: cutting Secret Bases frees exactly 3,200 B (+176 B
  decorations). No FREE_ config exists for it, so it's our own edit — mechanical,
  following the existing `FREE_*` #if pattern in `global.h`.
- **But SaveBlock1 headroom is only 304 B today** — the ~810 B habitat payload does
  not fit anywhere until something is freed. Reclamation is a prerequisite, not
  a nicety.
- **Discovery: SaveBlock3** — the expansion already maintains a spare-byte region
  spanning every save sector, sized 1,624 B, currently holding 4 B (two optional
  features, both off). The full §5 draft allocation (spot states 320 + talk
  counters 8 + plots 96 + registry 384 + faces 2 = 810 B) **fits in SaveBlock3
  today, doubled, with zero layout surgery** — it's the expansion's sanctioned
  place for new save data (`gSaveBlock3Ptr`, save/load chunking already handled).
- Total reclaimable ceiling if ever needed: ≈ 3.4 KB (Secret Bases + decor) +
  3.8 KB (FREE_ configs) + 3.6 KB (contest/TV/news/frontier) ≈ **10.8 KB** —
  ~13× the spec's draft need. The save budget is emphatically solved.

**Recommendation (needs your call, Jay — save layout is ask-first territory):**
put `struct HabitatSave` in **SaveBlock3** for the vertical slice (zero risk, zero
upstream diff, room for 810 B twice over), and plan the Secret Base cut as the
scale-out reservoir for the full 320-spot game (SB1 space also unlocks bigger
flag/var banks if story needs them). Alternative if you'd rather not split:
do the Secret Base cut now and put everything in SB1 from day one.

## Deviations from plan / risks

- Planned Lua-scripted verification was impossible on pinned mGBA (scripting CLI
  is Qt-only in 0.10.x); replaced with the libmgba-linked C runner. Better
  determinism, same evidence; slightly higher C maintenance cost, mitigated by
  the runner staying ~200 lines with one job.
- brew `arm-none-eabi-gcc` 16.1 was installed then removed (no newlib);
  Arm GNU Toolchain 14.2.Rel1 replaces it, matching upstream's proven GCC range.
- Cosmetic: running bare `make verify` without sourcing `env.sh` first prints six
  harmless `arm-none-eabi-gcc: command not found` lines while the outer Makefile
  parses (the script sources the env itself and builds fine). Documented here;
  fixable later by teaching the wrapper target to re-exec through env.sh.
- Linker warnings about "macOS-11.0 vs 26.0" dylibs during runner build: cosmetic
  version-floor mismatch, no functional impact observed.
